## UI 点击交互的正确实现方式

### 核心原则
**不要依赖 Pixi 的 `pointerdown` 事件**，在微信小游戏环境下不稳定。

统一使用**场景 `onTouchStart` + 命中检测**实现点击。

---

### 标准实现步骤

#### 1. 创建 UI 时记录点击区域
用 `{ x, y, w, h }` 记录命中区域,坐标使用设计分辨率（750x1334）。

#### 2. 在 `onTouchStart` 中做命中检测
- `getTouchPosition()` 已将物理像素转为设计分辨率坐标
- 用矩形命中检测判断点击区域
- 处理完立即 `return`，避免事件穿透

#### 3. 弹窗/暂停场景的处理
- 用状态标记（如 `isPaused`）控制交互优先级
- 在 `onTouchStart` 最前面检查状态，处理弹窗交互后直接 `return`
- 确保 `update()` 中也检查状态

---

### 注意事项
- 坐标统一使用**设计分辨率**（750x1334）
- **不要使用 `openModal()`**，它会导致 `InputManager` 完全不分发事件
- 用状态标记控制交互流程，不依赖输入拦截机制
